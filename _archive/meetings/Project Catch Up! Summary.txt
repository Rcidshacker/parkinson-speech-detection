Project overview — Parkinson's speech model goals
Mrudul presented the project objective to develop a speech-based foundation model for Parkinson’s disease with cross-language robustness, differentiation from related disorders (MSA/PSP), early prediction, and sex-aware analysis, motivated by speech changes preceding motor symptoms. The literature review noted most prior work is single-language or small cohorts.
* The proposed project aims to build a cross-language speech foundation model to detect Parkinson’s and distinguish similar disorders while supporting early and sex-aware analysis.

Dataset inventory and challenges
The team reviewed an inventory of datasets and metadata, noting roughly 10+ datasets and about 12,000 recordings spanning languages such as English, Spanish, Mandarin, Hindi, and Arabic. Challenges identified include varying sampling rates (8 kHz telephone vs high-resolution clinical audio), differing acquisition methods, metadata heterogeneity (medication status, UPDRS, MOCA), class imbalance, and limited early-detection labels.
* Several datasets exist (~10 datasets, ~12,000 recordings) but availability of original wave files and access restrictions (UCI, IEEE) must be verified.

Proposed methodology and model architecture
The group proposed a methodology of data acquisition and preprocessing (unify/resample to 16 kHz), voice-activity detection, automated acoustic feature extraction (MFCCs, jitter/shimmer, formants), multilingual transcription via Whisper for linguistic features, encoder training using CNN/Transformer architectures, and multi-class classification with leave-one-language-out cross-validation. Faculty asked for methodological clarifications and validation criteria.
* The proposed pipeline includes resampling to 16 kHz, voice-activity detection, acoustic and linguistic feature extraction, encoder training, and leave-one-language-out validation.

VAD, normalization, and feature-extraction tooling
The group recommended treating VAD as a feature extractor to capture pauses rather than removing silence, performing RMS/intensity normalization using active-speech RMS values, and using Librosa for broad feature extraction while acknowledging alternative libraries (e.g., Praat/Reprosody) provide different internal calculations.
* Voice activity detection should be used as a feature extractor (to capture pauses) and not to remove silence in preprocessing.
* All audio samples should receive RMS-level normalization to equalize intensity across recordings.
* Librosa was recommended as the primary feature-extraction library for reproducibility and breadth of functions.

Dataset categorization and model breakdown
The team agreed to categorize datasets by language and build language-specific models, then create gender-specific models independent of language, and finally create models based on speech types (vowels, sentences, repeated vowels, numbers) to study which features matter for each classification.
* Pramod will send a literature paper listing datasets and features to guide feature selection.
* Datasets should be categorized by language and then modeled separately for each language.
* Separate models should be built for gender-specific data regardless of language or speech type.
* Additional models should be developed by speech categorization (vowels, sentences, repeated vowels, numbers).

Foundation models and modeling choices
After initial experiments, two foundation-model strategies were proposed: (1) a model trained on handcrafted features and (2) a model using raw speech transformed to MEL spectrograms/STFT/chromograms fed to CNNs or transformers, with the architecture choice deferred until phase one results are available.
* Two foundation-model approaches are planned: (a) handcrafted features into a model and (b) raw speech transformed to spectrograms fed to CNN/transformer.

Baseline labeling and disease categorization guidance
A clarification was provided that Parkinson, MSA, and PSP are distinct but similar diseases, and the team should start with a binary healthy vs. non-healthy model before moving to multi-stage disease classification.
* Start with a binary classification (healthy vs. non-healthy) before attempting multi-stage classification.
